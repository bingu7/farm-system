# 第二阶段安全配置修复记录

本记录对应 `优化计划.md` 的“第二阶段：安全配置加固”。

本阶段遵循小步发布原则，只处理配置化和已确认的小范围安全项；登录失败次数限制、注册验证码等策略性功能暂不混入本次修改。

## 1. JWT 密钥配置化

涉及文件：

- `springboot/src/main/java/com/example/Utils/TokenUtils.java`
- `springboot/src/main/java/com/example/controller/WebController.java`
- `springboot/src/main/java/com/example/config/JwtInterceptor.java`
- `springboot/src/main/resources/application.yml`
- `springboot/src/main/resources/application-local.example.yml`
- `README.md`

问题：

- `TokenUtils` 原来使用硬编码默认密钥 `farm-system-jwt-secret`。
- 密钥不够长，也不利于不同环境独立配置。

解决方法：

- 将 `TokenUtils` 改为 Spring Bean。
- 从配置项 `jwt.secret` 读取 JWT 签名密钥。
- `application.yml` 使用环境变量 `JWT_SECRET` 覆盖：

```yaml
jwt:
  secret: ${JWT_SECRET:farm-system-local-dev-secret-change-me}
```

- 默认值只作为本地开发使用。
- `TokenUtils` 启动时检查密钥不能为空，且长度至少 32 个字符。
- `WebController` 和 `JwtInterceptor` 改为注入 `TokenUtils`，不再调用静态方法。
- `application-local.example.yml` 增加本地 `jwt.secret` 示例。
- `README.md` 增加 `JWT_SECRET` 配置说明。

验证方式：

- 后端编译通过。
- 登录生成 token 和拦截器验签仍使用同一份配置密钥。

## 2. 修改密码旧密码校验复查

涉及文件：

- `springboot/src/main/java/com/example/service/AdminService.java`
- `springboot/src/main/java/com/example/service/UserService.java`
- `springboot/src/main/java/com/example/controller/WebController.java`
- `vue/src/views/manager/Password.vue`

复查结果：

- 管理员和普通用户的 `updatePassword` 都已经要求 `password` 原密码不能为空。
- 后端会用 `PasswordUtils.matches(account.getPassword(), dbPassword)` 校验原密码。
- 原密码错误时返回 `原密码错误`。
- 前端修改密码页会提交 `password`、`newPassword`、`confirmPassword`。

处理方式：

- 本次不重复改动密码业务逻辑。
- 在记录中明确当前实现已经满足“修改密码要求验证旧密码”的第二阶段目标。

## 3. 运行配置分层

涉及文件：

- `springboot/src/main/resources/application.yml`
- `springboot/src/main/resources/application-local.example.yml`

问题：

- 默认配置中开启了 MyBatis `StdOutImpl` SQL 日志。
- SQL 直接输出到 stdout 更适合本地调试，不适合作为默认配置。

解决方法：

- 从 `application.yml` 移除：

```yaml
mybatis:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

- 将该配置放入 `application-local.example.yml`，需要本地查看 SQL 时再复制到真实 `application-local.yml`。
- `map-underscore-to-camel-case` 仍保留在默认配置中。

## 5. 暂不处理的第二阶段事项

以下内容暂不在本次实现：

- 登录失败次数限制。
- 注册频率限制或验证码。
- 强制生产环境未配置 JWT 密钥时启动失败。
- 数据库连接 SSL、HikariCP、Redis 连接池调优。

原因：

- 这些改动需要更多运行环境和前端交互策略配合。
- 本阶段先完成低风险配置加固，避免一次性扩大影响面。
