# 农产品供销系统接口文档

## 1. 基础信息

默认后端地址：

```text
http://localhost:9090
```

如果本地用其他端口启动，例如 `9091`，把文档中的地址替换为实际端口即可。

请求数据格式：

```http
Content-Type: application/json;charset=utf-8
```

登录后需要在请求头携带 token：

```http
token: 登录接口返回的 token
```

无需登录的接口：

- `POST /login`
- `POST /register`
- `GET /files/download/{fileName}`

其余接口默认都需要登录。

## 2. 通用响应格式

成功响应：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {}
}
```

失败响应：

```json
{
  "code": "400",
  "msg": "错误原因",
  "data": null
}
```

常见 HTTP 状态码：

| HTTP 状态码 | body.code | 说明 |
| --- | --- | --- |
| 200 | 200 | 请求成功 |
| 400 | 400 | 参数错误、业务校验失败 |
| 401 | 401 | 未登录、登录失效、token 错误 |
| 403 | 403 | 已登录但无权限 |
| 413 | 413 | 上传文件超过限制 |
| 500 | 500 | 服务器异常 |

## 3. 分页响应格式

分页接口统一返回 PageInfo 结构，常用字段如下：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {
    "list": [],
    "pageNum": 1,
    "pageSize": 10,
    "total": 100,
    "pages": 10
  }
}
```

## 4. 账号与系统接口

### 4.1 登录

```http
POST /login
```

权限：无需登录。

请求体：

```json
{
  "username": "admin",
  "password": "admin",
  "role": "ADMIN"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |
| role | string | 是 | `ADMIN` 管理员，其他值按普通用户登录 |

响应示例：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {
    "id": 1,
    "username": "admin",
    "name": "Bingu",
    "role": "ADMIN",
    "avatar": "http://localhost:9090/files/download/xxx.png",
    "token": "jwt-token"
  }
}
```

说明：登录响应不会返回密码字段。

### 4.2 注册普通用户

```http
POST /register
```

权限：无需登录。

请求体：

```json
{
  "username": "user001",
  "password": "123456",
  "name": "张三",
  "sex": "男",
  "phone": "13800138000",
  "email": "user001@example.com"
}
```

后端校验：

- 用户名只能包含字母、数字、下划线，长度 3 到 20 位。
- 密码长度 3 到 32 位。
- 手机号必须符合大陆手机号格式。
- 邮箱必须符合邮箱格式。

### 4.3 退出登录

```http
GET /logout
```

权限：登录用户。

说明：删除 Redis 中当前 token 的登录状态。

### 4.4 修改密码

```http
PUT /updatePassword
```

权限：登录用户。

请求体：

```json
{
  "password": "oldPassword",
  "newPassword": "newPassword"
}
```

说明：后端会根据当前登录用户自动判断管理员或普通用户。

### 4.5 统计数据

```http
GET /statistics
```

权限：登录用户。

说明：

- 管理员返回全站统计。
- 普通用户返回个人订单相关统计。

## 5. 文件接口

### 5.1 上传文件

```http
POST /files/upload
```

权限：登录用户。

请求格式：

```http
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| file | file | 是 | 上传图片 |

限制：

- 最大 5MB。
- 只允许图片：`jpg`、`jpeg`、`png`、`gif`、`webp`、`bmp`。

成功响应：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": "http://localhost:9090/files/download/文件名.png"
}
```

### 5.2 下载文件

```http
GET /files/download/{fileName}
```

权限：无需登录。

说明：返回图片二进制内容。

## 6. 管理员接口

基础路径：

```text
/admin
```

权限：全部需要管理员。

管理员字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 管理员 ID |
| username | string | 用户名 |
| password | string | 密码，新增/修改时可传 |
| name | string | 姓名 |
| avatar | string | 头像地址 |
| role | string | 固定为 `ADMIN` |

接口列表：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/admin/add` | 新增管理员 |
| DELETE | `/admin/delete/{id}` | 删除管理员 |
| PUT | `/admin/update` | 修改管理员 |
| GET | `/admin/selectById/{id}` | 根据 ID 查询 |
| GET | `/admin/selectAll` | 查询全部 |
| GET | `/admin/selectPage` | 分页查询 |

新增示例：

```json
{
  "username": "admin002",
  "password": "123456",
  "name": "管理员2",
  "avatar": "http://localhost:9090/files/download/avatar.png"
}
```

分页查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| pageNum | number | 1 | 当前页 |
| pageSize | number | 10 | 每页数量 |
| username | string | 无 | 可选筛选 |
| name | string | 无 | 可选筛选 |

## 9. 商品接口

基础路径：

```text
/goods
```

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 商品 ID |
| name | string | 商品名称 |
| img | string | 商品图片地址 |
| descr | string | 商品描述 |
| specials | string | 商品特色 |
| price | number | 商品价格 |
| unit | string | 单位 |
| store | number | 库存 |
| categoryId | number | 分类 ID |
| categoryName | string | 分类名称，查询时返回 |

接口列表：

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| POST | `/goods/add` | 管理员 | 新增商品 |
| DELETE | `/goods/delete/{id}` | 管理员 | 删除商品 |
| PUT | `/goods/update` | 管理员 | 修改商品 |
| GET | `/goods/selectById/{id}` | 登录用户 | 根据 ID 查询 |
| GET | `/goods/selectAll` | 登录用户 | 查询全部 |
| GET | `/goods/selectPage` | 登录用户 | 分页查询 |

新增示例：

```json
{
  "name": "苹果",
  "img": "http://localhost:9090/files/download/apple.png",
  "descr": "新鲜苹果",
  "specials": "产地直发",
  "price": 9.9,
  "unit": "斤",
  "store": 100,
  "categoryId": 1
}
```

分页查询参数：

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| pageNum | number | 1 | 当前页 |
| pageSize | number | 10 | 每页数量 |
| name | string | 无 | 按商品名模糊查询 |
| categoryId | number | 无 | 按分类筛选 |

后端校验：

- 商品名称不能为空。
- 商品价格不能小于 0。
- 商品库存不能小于 0。
- 商品分类不能为空。

删除保护：

- 商品已有订单时，不能删除。
- 商品已有库存记录时，不能删除。

## 11. 订单接口

基础路径：

```text
/orders
```

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 订单 ID |
| orderNo | string | 订单号，后端生成 |
| goodsId | number | 商品 ID |
| num | number | 购买数量 |
| userId | number | 下单用户 ID，后端按当前登录用户设置 |
| status | string | 订单状态 |
| time | string | 下单时间，后端生成 |
| goodsName | string | 商品名称，查询时返回 |
| goodsImg | string | 商品图片，查询时返回 |
| userName | string | 用户名称，查询时返回 |

接口列表：

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| POST | `/orders/add` | 登录用户 | 创建订单 |
| DELETE | `/orders/delete/{id}` | 管理员 | 删除订单 |
| PUT | `/orders/update` | 管理员或订单本人 | 修改订单状态 |
| GET | `/orders/selectById/{id}` | 管理员或订单本人 | 根据 ID 查询 |
| GET | `/orders/selectAll` | 登录用户 | 查询订单；普通用户只能看自己的订单 |
| GET | `/orders/selectPage` | 登录用户 | 分页查询；普通用户只能看自己的订单 |

创建订单示例：

```json
{
  "goodsId": 1,
  "num": 2
}
```

说明：

- `orderNo`、`userId`、`status`、`time` 由后端生成或覆盖。
- 下单成功后会扣减商品库存。
- 购买数量必须大于 0。
- 库存不足会返回错误。

修改订单状态示例：

```json
{
  "id": 1,
  "status": "待发货"
}
```

订单状态流转规则：

| 操作人 | 允许流转 |
| --- | --- |
| 普通用户 | `待支付` -> `待发货` |
| 普通用户 | `待支付` -> `已取消` |
| 普通用户 | `待收货` -> `已完成` |
| 管理员 | `待发货` -> `待收货` |

取消订单说明：

- 订单改为 `已取消` 时，后端会恢复商品库存。

## 13. 查询参数补充

多数 `selectAll` 和 `selectPage` 接口支持把实体字段作为 query 参数筛选。

示例：

```http
GET /goods/selectPage?pageNum=1&pageSize=10&name=苹果&categoryId=1
```

```http
GET /orders/selectPage?pageNum=1&pageSize=10&orderNo=abc
```

```http
GET /notice/selectPage?pageNum=1&pageSize=10&title=公告
```

## 14. 调用示例

### 14.1 登录后查询商品

```bash
curl -X POST "http://localhost:9090/login" \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"admin\",\"password\":\"admin\",\"role\":\"ADMIN\"}"
```

拿到 token 后：

```bash
curl "http://localhost:9090/goods/selectPage?pageNum=1&pageSize=10" \
  -H "token: your-token"
```

### 14.2 上传图片

```bash
curl -X POST "http://localhost:9090/files/upload" \
  -H "token: your-token" \
  -F "file=@D:/test/apple.png"
```

## 15. 重要业务说明

- 密码已使用哈希存储，接口响应不会返回密码字段。
- 普通用户不能越权查看或修改其他用户资料。
- 普通用户不能查看其他用户订单。
- 管理员才能管理用户、管理员、商品、分类、公告、入库记录。
- 商品、用户、分类删除前会检查关联数据，避免历史订单或商品信息断链。
- 本地敏感配置建议写入 `application-local.yml`，不要提交到 GitHub。
