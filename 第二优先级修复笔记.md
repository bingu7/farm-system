# 第二优先级修复笔记

本次修复围绕三个点：上传大小限制、HTTP 状态码规范、后端表单输入校验。

## 1. 上传大小限制

### 原问题

原来 Spring 上传限制是 100MB。对头像、商品图片来说太大，容易带来这些问题：

- 用户误传大文件时，占用后端带宽、内存和磁盘。
- 前端等待时间长，体验差。
- 后端虽然有限制文件类型，但缺少业务层大小校验。

### 修改内容

配置文件中把 multipart 上传限制改成 5MB：

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 5MB
      max-request-size: 5MB

file:
  max-upload-size: ${FILE_MAX_UPLOAD_SIZE:5242880}
```

这里 `5242880` 是 5 * 1024 * 1024，也就是 5MB。

`FileController` 中增加了二次校验：

- 空文件拒绝上传。
- 文件大小超过 `file.max-upload-size` 拒绝上传。
- 只允许图片后缀：`jpg`、`jpeg`、`png`、`gif`、`webp`、`bmp`。
- 只允许图片 Content-Type。

前端 `vue/src/utils/upload.js` 也增加了 5MB 检查，用户选择文件时就能提前提示。

## 2. HTTP 状态码规范

### 原问题

以前很多错误虽然 body 里写了 `code: "401"`、`code: "403"`，但 HTTP 状态码仍然是 200。这样调试工具、前端拦截器、部署监控都不容易准确判断错误类型。

### 修改内容

全局异常处理改成返回真实 HTTP 状态码：

| 业务 code | HTTP 状态码 | 含义 |
| --- | --- | --- |
| `400` | 400 | 参数错误、表单校验失败 |
| `401` | 401 | 未登录、登录失效、token 错误 |
| `403` | 403 | 已登录但无权限 |
| `413` | 413 | 上传文件过大 |
| `500` | 500 | 后端未知异常 |

`GlobalExceptionHandler` 现在返回 `ResponseEntity<Result>`，既保留原来的统一 JSON body，又让 HTTP 状态码更标准。

前端 `vue/src/utils/request.js` 也做了适配：

- HTTP 401：提示后清理本地登录信息，并跳回登录页。
- HTTP 403：提示无权限。
- 其他非 200 业务 code：显示后端返回的错误信息。
- HTTP 错误响应也会读取 body，不再只显示笼统的网络异常。

上传组件增加了 `on-error` 处理，能显示后端返回的上传错误信息。

## 3. 后端表单输入校验

### 原问题

前端校验只能改善体验，不能作为安全边界。用户可以绕过前端直接调接口，所以后端也必须做硬校验。

### 修改内容

新增 `ValidationUtils`，统一处理常用校验：

- 用户名不能为空。
- 用户名只能包含字母、数字、下划线，长度 3 到 20 位。
- 密码不能为空，长度 3 到 32 位。
- 手机号格式校验。
- 邮箱格式校验。

在服务层增加校验：

- `AdminService`：新增、修改、登录、改密码时校验用户名和密码。
- `UserService`：新增、修改、注册、登录、改密码时校验用户名、密码、手机号、邮箱。
- `GoodsService`：新增、修改商品时校验商品名、价格、库存、分类。
- `OrdersService` 之前已处理下单数量必须大于 0，本次继续沿用。

商品校验规则：

- 商品名称不能为空。
- 商品价格不能小于 0。
- 商品库存不能小于 0。
- 商品分类不能为空。

## 4. 本次验证结果

后端编译：

```bash
mvn -q -DskipTests compile
```

结果：通过。

前端构建：

```bash
npm run build
```

结果：通过。

接口实测结果：

| 场景 | 结果 |
| --- | --- |
| 未登录访问 `/user/selectPage` | HTTP 401，`code: "401"`，提示“请登录” |
| 空用户名登录 | HTTP 400，`code: "400"`，提示“用户名不能为空” |
| 注册时邮箱格式错误 | HTTP 400，`code: "400"`，提示“邮箱格式不正确” |
| 管理员登录 `admin/admin` | HTTP 200，登录成功并返回 token |
| 新增商品时价格为 -1 | HTTP 400，`code: "400"`，提示“商品价格不能小于0” |
| 上传 txt 文件 | HTTP 400，`code: "400"`，提示只支持图片 |
| 上传超过 5MB 图片 | HTTP 413，`code: "413"`，提示“上传文件不能超过5MB” |

## 5. 后续注意

目前仍有两个后续优化点：

- 可以逐步把更多实体改成 Java Bean Validation，例如 `@NotBlank`、`@Min`、`@Email`，让校验规则更声明式。
- 前端构建有 chunk size warning，说明打包后的部分文件较大，后面可以做按需加载或手动分包优化。
